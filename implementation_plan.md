# ClaimIQ MVP — Implementation Plan (v3 Final Aligned)

## Goal

Build a working end-to-end demo: upload healthcare claim PDFs → AI classifies, extracts, cross-checks, reasons → dashboard shows readiness score, financial impact, issues, and fix recommendations. Buildable in ~3.5 hours.

---

## Aligned Design Decisions (via `/grill-me`)

| Decision Area | Chosen Approach | Rationale |
|---|---|---|
| **Frontend Tech Stack** | **React + Vite** | Component-based architecture, premium enterprise SaaS aesthetic, dynamic state management for multi-stage animations. |
| **Styling & Icons** | **Vanilla CSS + `lucide-react`** | Sleek dark mode (#0f1724), glassmorphism cards, custom gradients, smooth keyframe animations, Inter/Outfit Google Fonts. |
| **API Call Strategy** | **5-Call DeepSeek Pipeline** | Dedicated calls for classification/quick extraction, deep bill/discharge extraction, medical reasoning, financial audit, and summary. Highest audit accuracy. |
| **Real-time Progress** | **Server-Sent Events (SSE)** | Lightweight streaming from FastAPI (`sse-starlette`) to React as each processing stage completes. Animates the 6-stage pipeline smoothly. |
| **Demo Pitching UX** | **'⚡ Load Demo Claim Packet' Button** | One-click button on the upload dropzone that auto-loads the 6 generated sample PDFs instantly during live pitching. |

---

## What We Borrowed from AarogyaID

1. **Named Live Processing Stages** — Show our pipeline as 6 live AI steps with time elapsed and status messages.
2. **Per-Document Audit Cards** — Display extracted data, document type, confidence badges, and status icons.
3. **Claims Intelligence Summary** — Paragraph summary written in an auditor's voice.
4. **Financial Impact Quantification** — Highlight "₹12,400 at risk out of ₹85,000 claimed" with itemized risk breakdown.
5. **Confidence-Based HITL Flags** — Mark uncertain AI items as **"Needs Human Review"**.
6. **Claim Timeline Visualization** — Visual timeline highlighting date conflicts inline.

---

## DeepSeek Call Budget (5 Calls)

| Call # | Purpose | Input | Output |
|---|---|---|---|
| **1** | **Batch Classify + Quick Extract** | All doc texts truncated (~1000 chars each) | JSON mapping doc types + basic entities (names, dates, totals) |
| **2** | **Deep Extraction (Bill + Discharge)** | Full text of Hospital Bill & Discharge Summary | Detailed line items, medications, ICD codes, doctor registration numbers |
| **3** | **Medical Reasoning Audit** | All extracted clinical entities | Diagnosis ↔ procedure mismatches, unindexed medications, upcoding checks |
| **4** | **Financial Audit** | Bill line items + amounts across all docs | Overcharges, duplicate billing, missing receipts, arithmetic mismatches |
| **5** | **Claims Intelligence Summary** | All combined issues + scores | Conversational executive briefing, prioritized fixes, financial risk estimation |

## Build Order & Timeline (~3.5 Hours)

1. **Sample Data Generator (`generate_samples.py`)** — Create 6 rich PDFs with hospital header, doctor registration numbers, and planted bugs. *(20 min)*
2. **Backend Infra (`utils.py`, `ocr_engine.py`)** — Port fuzzy matching & normalization from `comparator.py`, setup Docling OCR. *(15 min)*
3. **AI Processing Layer (`document_processor.py`, `cross_checker.py`)** — Wire up the 4 DeepSeek analytical calls + deterministic comparison rules. *(45 min)*
4. **Scoring & Reporting (`scorer.py`, `report_generator.py`)** — Score math + DeepSeek final summary generator. *(20 min)*
5. **API & SSE Server (`main.py`)** — Endpoints for upload, demo loading, and real-time SSE streaming. *(25 min)*
6. **React Frontend Setup & Components** — Initialize Vite React app, build sleek glassmorphism UI components (`UploadZone`, `PipelineProgress`, `Dashboard`). *(60 min)*
7. **End-to-End Testing & Polish** — Verify demo button flow, animation timing, and DeepSeek responses. *(25 min)*
